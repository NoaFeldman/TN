64 // NB! sign convention on CGCs // tag: CGC_NORM
65 //
66 // using CRef, ensure that CData is simply normalized to 1
67 // internally within gCG. This ensures that for each OM set
68 // * overlaps can simply be computed based on cgw data!
69 // * OM within gCG can be also determined through cdata.norm2().
70 // return value: number of trailing dimensions to be interpreted
71 // as OM space.
72 //
73 // Furthermore, keep |cgw|^2=1 as far as possible,
74 // so A.norm2() of some QSpace A is simply carried by |A.data|^2.
75 // @QSpace/display.m also assumes this convention throughout.
76 //
77 // => data already carries correct 3j weight factors
78 // for simple contraction [e.g. norm2()]
79 // => identity A tensors have entries sqrt(d_out)
80 // which already indicates multiplet weight
81 //
82 // Other (not so useful) attempts to normalize cgw
83 // (with corresponding inverse factor multiplied onto data{:}):
84 //
85 // * |cgw| := sqrt(max(size(cdata))) // see also CGC_NORM_EXT
86 //
87 // (+) rank-2 tensor are exact identities (or untaries for 1j symbols)
88 // (+) rank-3 tensors are mostly normalized to max(|CGC|)
89 // e.g. (SS|0) is normalized the same way as (S0|S)
90 // whereas their standard normalization [see NormStd()]
91 // would be sqrt(d_out), here 1 or (2S+1), respectively!
92 //
93 // (-) standard rank-3 identity A tensors do not have cgr = Id
94 // (-) does not work specifialy well for rank>3 tensors.
95 // e.g. consider
96 // contract( (1,1|2), 3, (1,1|2), 3 ) (i)
97 // vs. contract( (1,1|0), 3, (1,1|0), 3 ) (ii)
98 // while the output CData space is the same (1,1,1,1)
99 // in either case, already the normalization of the
100 // CData requires *different* normalization factors 1/sqrt(d3)

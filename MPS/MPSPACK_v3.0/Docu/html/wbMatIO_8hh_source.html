51  // that can store objects that occupy more than 2GB(!)
52  // upon reading, mat-type is determined automatically
53  // NB! w7.3 creates some extra overhead, though
54  // eg. a 22MB files becomes 27MB (21,576,744 => 27,314,288 bytes)
55  // Wb,Feb10,11
56  // ------------------------------------------------------------ //
57  // however, if the data can be compressed, *regular* mat-file
58  // format produces significantly smaller (compressed!) files
59  // e.g. a 22kB byte structure (w7.3) => becomes 600 bytes!
60  // e.g. a 300 MB file (w7.3) => becomes 100MB (similar reduction

62  // see also "wz" flag below! // Wb,Jan25,15

69  // also w7.3 allows to use matlab's matfile()

73  else if (!strcmp(RW,"w:std")) {
74  // NB! 'wz' enables compressed written data!
75  // similar to what matlab desktop / command line does!
76  // (otherwise read/rewrite results in about clearly smaller
77  // files, e.g. 1/4 the size written with plain "w" flag!)
